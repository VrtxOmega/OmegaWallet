# Support

OmegaWallet is a pre-release, open-source project maintained by the VERITAS & Sovereign Ecosystem team. Support is provided on a best-effort basis.

---

## Before Asking for Help

- Check the [README](./README.md) — quickstart, configuration, and architecture are documented there.
- Check [KNOWN_LIMITATIONS.md](./KNOWN_LIMITATIONS.md) — your issue may already be a documented limitation.
- Search [open and closed issues](https://github.com/VrtxOmega/OmegaWallet/issues) to see if it has been reported before.

---

## Reporting a Bug

Open a [GitHub Issue](https://github.com/VrtxOmega/OmegaWallet/issues/new) and include:

- OmegaWallet version (e.g., `5.0.0-rc1`)
- Operating system and version
- Steps to reproduce
- What you expected to happen
- What actually happened
- Relevant logs (redact any private keys, wallet addresses you care about, or RPC keys)

---

## Reporting a Security Vulnerability

**Do not open a public issue for security vulnerabilities.**

Use the private disclosure process described in [SECURITY.md](./SECURITY.md).

---

## Feature Requests

Open a [GitHub Issue](https://github.com/VrtxOmega/OmegaWallet/issues/new) with the `enhancement` label. Include:

- Clear description of the feature
- Use case and motivation
- How it fits within the renderer-cannot-sign trust model

Feature requests that would weaken the IPC boundary or introduce key exposure in the renderer will not be considered.
